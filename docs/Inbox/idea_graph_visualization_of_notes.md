# アイデア：フリー記述メモのグラフビュー可視化 (Obsidian風)

## コンセプト
トレーニング記録に含まれる「フリー記述（メモ）」や「タグ」を、Obsidianのグラフビューのようにネットワーク状に連結表示する機能。

## 目的・提供価値
従来のアプリは「重量×回数」の**定量的データ**の分析には強いが、ユーザーの「感覚」や「コンディション」といった**定性的データ**の活用が弱い。
この機能により、以下のような相関関係を**視覚的**に発見可能にする。

*   **コンディションとパフォーマンスの相関:**
    *   「寝不足」という単語が含まれる日 ⇔ 「ベンチプレス」の重量低下 ⇔ 「関節痛」
*   **サプリメント/食事の効果:**
    *   「カフェイン」 ⇔ 「集中力高」 ⇔ 「PR更新」
*   **相性の良い組み合わせ:**
    *   特定の「種目」と特定の「ウォームアップ内容」のつながり

## ユーザー体験 (UX)
1.  **入力:** ユーザーは普段通り、トレーニング中に自由にメモ（例：「今日は膝が少し痛い」「昨日の焼肉でパワーある」）を取る。
2.  **処理:** アプリが自然言語処理（または単純なタグ抽出）でキーワードを抽出。
3.  **可視化:**
    *   ノード（点）： 「単語（膝痛）」、「種目（スクワット）」、「部位（脚）」
    *   エッジ（線）： 同じ日に記録された、または頻繁に共起する関係
4.  **探索:** ユーザーはグラフを操作し、特定の単語（例：「腰痛」）を中心にどのような要素が結びついているかを探る。

## 技術的アプローチ
*   **可視化ライブラリ:** D3.js, Cytoscape.js, React Force Graphなど。
*   **データ構造:** ノードとリンクを持つグラフ構造としてデータを整形。
*   **NLP (自然言語処理):** 簡易的な形態素解析で名詞・形容詞を抽出、あるいはハッシュタグ（#）ベースからスタートする。
